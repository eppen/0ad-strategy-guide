# 多人对战

## 联机方式

| 方式 | 说明 |
|------|------|
| **Lobby** | 官方大厅，XMPP；R28 默认 TLS 验证 |
| **Connect by IP** | 直连 / 原 LAN 入口改名 |
| **Hotseat** | 本地同机轮流 |
| **Replay / Observer** | 观战与复盘 |

- 最多 **8 人**；Skirmish、Deathmatch、Relic 等模式。
- **P2P** 同步；注意 NAT——R28 简化 STUN，多数情况无需手动端口转发。
- **Ranked**：胜负影响 rating；选熟悉文明与地图。

## 与 AI 战的差异

1. **不会等你的经济** — Town 慢 = 被 rush。
2. **侦察反侦察** — 藏 CC、假 dropsite、看小地图动向。
3. **兵种针对性** — 见敌方 Barracks 产兵调整 counter。
4. **断经济** — 杀 Trader、烧 Farmstead 往往比硬攻 CC 更有效。

## 常见开局套路

### Fast Town（稳健）

- 标准 5 建筑 + dropsite，7 分钟内 Town，塔防过渡。
- 适合罗马、雅典、汉等 scaling civ。

### Early Pressure（施压）

- Spartans / Gauls：少平民多兵，Town 前骚扰 citizen。
- 风险：被防住后经济落后。

### Fast City（冒险）

- 队友掩护或 1v1 封闭地图；拼 City 时间差。
- 需要金属矿近、少骚扰。

### Relic 模式

- 早兵占 relic 点；需 priest/驻防与机动增援。
- 与标准 destroy 节奏不同，勿纯憋 late。

## 团队战（2v2 / 3v3）

- **分工**：一 economy boom，一 early military；或双 boom + 队友 anti-rush。
- **共享视野**（若规则允许）协调 gank。
- **文明互补**：迦太基 merc + 罗马 infantry；汉人口 + 马其顿 siege。
